# 이슈 작성 가이드

## 제목 작성 기준

- 명사형으로 간결하게
- 기술 구현보다 기능 중심으로

**좋은 예:**
- "회원 프로필 조회 API 구현"
- "SMS 인증 만료 처리 추가"
- "로그인 실패 시 잠금 기능 구현"

**나쁜 예:**
- "MemberReader에 findByMemberKey 메서드 추가"
- "fix bug"

---

## 본문 템플릿

```markdown
## 개요
{구현하려는 기능 또는 해결하려는 문제를 1~2문장으로 설명}

## 배경
{왜 이 기능이 필요한지, 어떤 상황에서 요청되었는지}

## 완료 조건
- [ ] {구체적인 완료 기준 1}
- [ ] {구체적인 완료 기준 2}

## 참고
{관련 API 문서, 이슈, PR 링크 등}
```

---

## 작성 예시

**제목**: 회원 탈퇴 API 구현

**본문**:

```markdown
## 개요
회원이 서비스를 탈퇴할 수 있는 API를 구현한다.

## 배경
현재 탈퇴 기능이 없어 회원 데이터를 직접 삭제해야 하는 문제가 있음.

## 완료 조건
- [ ] DELETE /api/v1/members/me 엔드포인트 구현
- [ ] 소프트 딜리트 적용 (deletedAt 세팅)
- [ ] 탈퇴 시 RefreshToken 삭제
- [ ] 단위 테스트 작성

## 참고
CLAUDE.md - Soft Delete 전략 섹션
```
