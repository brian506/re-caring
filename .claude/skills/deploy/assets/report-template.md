# 완료 보고 템플릿

파이프라인 완료 후 아래 형식으로 사용자에게 보고한다.

---

## 배포 완료 보고 형식

```
## 배포 완료

**이슈**: #{번호} - {제목}
**PR**: {PR URL}
**커밋**: {커밋 메시지}

**결과**
- 빌드/테스트: 통과
- CI 체크: 통과
- PR 머지: 완료
- 배포: 성공
- 헬스체크: UP (re-caring.duckdns.org)
- 브랜치 정리: feature/{N} 삭제 완료
```

---

## 실패 보고 형식

실패 시 어느 단계에서 멈췄는지 명확히 표시한다.

```
## 배포 중단 — {실패 단계}

**이슈**: #{번호} - {제목}
**실패 단계**: {Step N — 단계 이름}
**원인**: {오류 요약}

**시도 내역**
1. {첫 번째 시도 내용}
2. {두 번째 시도 내용}
3. {세 번째 시도 내용}

**현재 상태**
- 브랜치: feature/{N} (삭제되지 않음)
- PR: {URL} ({상태})

**다음 조치 필요**
{사용자가 직접 확인해야 할 사항}
```
