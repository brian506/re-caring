> 최초 작성: 2026-04-15
> 기준 문서: 대상자_기능명세서_1차 (PDF) + 팀 요구사항 확정본

---

## 1. 프로젝트 개요

- **앱명**: Re;caRing
- **목적**: 보호자가 보호 대상자의 위치·이상상황을 실시간으로 모니터링하는 케어 앱

---

## 2. 사용자 역할 (MemberRole)

| Role | 설명 |
|------|------|
| `GUARDIAN` | 보호자 — 대상자의 위치/상태 모니터링 |
| `PATIENT` | 보호 대상자 — 위치를 보호자에게 공유 |

- 역할은 **회원가입 시 1회 선택**, 이후 변경 불가
- 역할에 따라 로그인 후 진입하는 메인 화면이 분기됨

---

## 3. 인증/계정 정책

### 3-1. 회원가입 플로우

#### 로컬 가입
```
역할 선택(SU-001) → 휴대폰 인증(SU-002) → 이메일 입력(SU-003)
  → 비밀번호 입력(SU-004) → 프로필 설정(SU-005)
  → 이용약관 동의(SU-006) → 가입 완료(SU-007)
```

#### 소셜 가입 (카카오/네이버)
- 소셜 인증 → 프로필 설정 → **신규 Member 생성**
- 소셜 계정으로 처음 진입한 경우 자동 회원가입 처리

#### 기존 로컬 가입자의 소셜 연동
- **휴대폰 인증(SMS 토큰)** 으로 기존 회원 검증 후 소셜 계정 연동

### 3-2. 비밀번호 정책

> ⚠️ PDF의 "6자 이상" 기준은 **아래 정책으로 대체**

- 영문 포함 **8자 이상**
- 특수문자 불필요

### 3-3. 로그인

- **Access Token**: JWT, 응답 바디(`accessToken` 필드)로 전달
- **Refresh Token**: JWT, `HttpOnly Cookie`로 전달

### 3-4. 아이디(이메일) 찾기

- 입력: 이름 + 생년월일 + 전화번호
- 출력: **마스킹 처리된 이메일**을 앱 화면에서 직접 표시
  - 예: `ho***@naver.com`
- 별도 이메일 발송 없이 앱 내 화면에서 바로 보여줌

### 3-5. 비밀번호 찾기

- **SMS 인증 토큰**으로 회원 검증 → 새 비밀번호 설정

## 4. 멤버십 정책

### BASIC (기본)

- 보호자 1명 → **대상자 최대 1명** 관리

### PREMIUM (프리미엄)

- 보호자 1명 → **대상자 여러 명** 관리 가능
- 각 대상자별 **추가 보호자(보호자2) 최대 1명** 설정 가능
- **관리자/관계자 최대 5명** 추가 가능

#### 독립 구독 케이스
- 보호자2가 별도로 프리미엄 구독 시 → 자신의 구독 기준으로 **독립적으로 다른 대상자 관리 가능**
- 구독은 계정 단위로 적용되며, 관계에 따라 권한이 공유되지 않음

### 관계 구조 요약

```
보호자(GUARDIAN, 구독자)
  └── 대상자1(PATIENT)
  │     ├── 추가보호자(보호자2, 최대 1명)
  │     └── 관리자/관계자 (최대 5명)
  └── 대상자2(PATIENT)  ← PREMIUM only
  └── 대상자N...        ← PREMIUM only
```
